────────────────────────────────────────

────────────────────────────────────────

NAVIGATION (sticky, both pages)
────────────────────────────────────────
- Horizontal, sticky to top
- Background: transparent with ~15% black overlay (use --overlay-dim) and backdrop-blur-sm
- Left: logo (placeholder SVG for now)
- Right: Home · Projects · Contact (Projects scrolls to section on /, navigates to /#projects from /contact)
- On mobile: same horizontal layout, no hamburger — keep it minimal

────────────────────────────────────────
SCROLL CHOREOGRAPHY (the key effect)
────────────────────────────────────────
Each section overlays the next as the user scrolls. Implementation:
- Wrap each section in a sticky positioned container
- As the user scrolls down, the *next* section translates up from below (translateY: 100% → 0) and covers the current one
- Scrolling back up reverses cleanly — the covering section slides back down, revealing the previous section already in place underneath (NOT re-animating in)
- Use GSAP ScrollTrigger with scrub: true so the animation is tied directly to scroll position
- Lenis for smooth scroll on desktop; native scroll on mobile

────────────────────────────────────────
CONTENT FADE-INS
────────────────────────────────────────
- Text, images, and UI elements fade + translate up 16px on first entry into viewport
- Stagger children by 60–80ms within a section
- Plays once per element — do not re-animate on scroll back up
- Background textures are exempt (no fade)
